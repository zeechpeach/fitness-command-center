Dynamic Workout Types - Implementation Test

Date: February 18, 2026

This document demonstrates the implementation of fully customizable workout programs where workout types are determined by user-defined day names.

✓ Core Changes Implemented

  • Removed hardcoded workout type cycling (Upper, Lower, Push, Pull, Legs, Rest)
  • New programs now start with completely blank slate
  • Workout type automatically set to day name
  • Home page displays dynamic buttons: "Day X (DayName)"
  • Clicking schedule pills prompts for day name (no cycling)

1. New Program Creation

Before (Old Behavior):

New Program {
  schedule: {
    day1: { workoutType: 'Rest', customName: '' },
    day2: { workoutType: 'Rest', customName: '' },
    day3: { workoutType: 'Rest', customName: '' },
    day4: { workoutType: 'Rest', customName: '' },
    day5: { workoutType: 'Rest', customName: '' },
    day6: { workoutType: 'Rest', customName: '' },
    day7: { workoutType: 'Rest', customName: '' }
  },
  workouts: {
    Rest: [{ name: 'Rest Day Recovery', sets: 1, reps: 'Complete' }]
  }
}

After (New Behavior):

New Program {
  schedule: {},  // Completely blank!
  workouts: {}   // No preset workouts!
}

Result: Users must define their own days and workout types from scratch.

2. Adding Days and Naming Them

User Action:

  1. Click "+" to add a day (e.g., Day 1)
  2. Click on the day pill or edit button
  3. Enter name: "Chest & Triceps"

Result:

schedule: {
  day1: {
    workoutType: 'Chest & Triceps',  // = day name
    customName: 'Chest & Triceps'    // = day name
  }
},
workouts: {
  'Chest & Triceps': []  // Ready for exercises
}

Key Change: No separate type selection - the name IS the type!

3. Home Page Dynamic Buttons

Example Program:

schedule: {
  day1: { workoutType: 'Chest Day', customName: 'Chest Day' },
  day2: { workoutType: 'Back Day', customName: 'Back Day' },
  day3: { workoutType: 'Leg Day', customName: 'Leg Day' },
  day4: { workoutType: 'Rest', customName: 'Rest' }

Home Page Displays:

┌─────────────────┐  ┌─────────────────┐  ┌─────────────────┐  ┌─────────────────┐
│ Day 1           │  │ Day 2           │  │ Day 3           │  │ Day 4           │
│ (Chest Day)     │  │ (Back Day)      │  │ (Leg Day)       │  │ (Rest)          │
└─────────────────┘  └─────────────────┘  └─────────────────┘  └─────────────────┘

Dynamic: Button labels automatically generated from program structure!

4. Workout Logging Flow

When User Selects "Day 1 (Chest Day)":

  1. System reads: day1.workoutType = 'Chest Day'
  2. Loads exercises from: workouts['Chest Day']
  3. When workout is saved, workoutType: 'Chest Day' is stored

No Manual Selection: Workout type is automatically determined!

6. Backward Compatibility

Existing programs (like ULPPL) continue to work:

schedule: {
  day1: { workoutType: 'Upper', customName: '' },
  day2: { workoutType: 'Lower', customName: '' },
  ...
}
workouts: {
  'Upper': [...exercises...],
  'Lower': [...exercises...],
  ...
}

Migration: Old programs function normally. Users can edit them to add custom names if desired.

7. Code Changes Summary

Modified Functions:

  • openProgramEditor() - Initialize with empty schedule/workouts
  • updateCycleLength() - New days start with empty name
  • editProgramDayName() - Sets both workoutType and customName to same value
  • selectWorkoutForDay() - Now calls editProgramDayName instead of cycling
  • getWorkoutTypeForDay() - Returns empty string instead of 'Rest' default
  • getDisplayNameForDay() - Shows "(click to name)" for unnamed days
  • renderWorkoutDaySelector() - Formats as "Day X (DayName)"
  • renderSchedulePills() - Shows helpful message for empty programs
  • renderWorkoutsAccordion() - Skips unnamed days, shows message if empty

Removed Hardcoding:

  • Removed: const workoutTypes = ['Rest', 'Upper', 'Lower', 'Push', 'Pull', 'Legs']
  • Removed: Cycling through predefined types
  • Removed: Default 7-day Rest schedule for new programs
